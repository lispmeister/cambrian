## Phase 1: Fitness Vector Extension (cambrian-mw0)

Three of the 15 fitness dimensions are self-referential: `test_count`, `test_pass_rate` (and hypothetically `test_coverage`). Under selection pressure a spec can evolve to describe code with 500 `assert True` tests and trivial implementations. Research ֲ§7 of the adversarial review flagged this and proposed test-quality mitigations.

### Changes to `test-rig/test_rig.py`

Added `_analyze_test_quality(test_file_paths)` helper:
- `_ASSERT_RE`: matches any `assert` statement per test file
- `_TRIVIAL_ASSERT_RE`: matches `assert True/False/None` patterns
- `_TEST_FUNC_RE`: counts `def test_` / `async def test_` functions

New fitness dimensions emitted by `compute_fitness()`:

| Dimension | Type | Description |
|-----------|------|-------------|
| `assertion_density` | float | assertions / test functions (0.0 if no functions) |
| `trivial_assert_rate` | float | trivial asserts / total asserts (lower is better) |
| `fitness_weights` | dict | discount weights for self-referential dims (0.5 for `test_count`, `test_pass_rate`, `trivial_assert_rate`) |

Updated BOOTSTRAP-SPEC-002 ֲ§2.8 to document test-quality dimensions and the discount weight semantics. No version bump taken yet (additive, backward-compatible ג€” will ask before bumping).

## Phase 2: Campaign Runner (cambrian-9ic)

New module `supervisor/campaign.py`.

### `compute_campaign_summary(records)`

Pure function. Input: list of generation records (as stored in `generations.json`). Output: `CampaignSummary` dict matching CAMBRIAN-SPEC-005 ֲ§Campaign:

| Field | Computation |
|-------|-------------|
| `viability_rate` | viable / total |
| `fitness_mean` | mean of each numeric fitness key across all records |
| `fitness_trend` | closed-form linear regression slope on [0,1] viability outcomes |
| `failure_distribution` | `{failure_stage: count}` |
| `stages_completed_distribution` | `{max_stage_reached: count}` |
| `generation_count` | total records in campaign |

`_linear_slope(values)`: closed-form least-squares (n*־£xy גˆ’ ־£x*־£y) / (n*־£xֲ² גˆ’ (־£x)ֲ²). Returns 0.0 for < 2 points.
`_max_stage(stages_completed)`: finds the furthest stage in `_STAGE_ORDER`.

### `run_campaign(spec_path, n, supervisor_url)`

Async orchestration. Generates a `campaign-<8hex>` ID, hashes the spec file, then runs N sequential generations via `POST /spawn` + polling `GET /versions`. Returns the CampaignSummary augmented with `campaign_id` and `spec_hash`.

**Test results:** 13 new tests across `TestComputeCampaignSummary` and `TestLinearSlope`, 214 total pass.

## Phase 3: Spec Diff Tooling (cambrian-evw)

New module `supervisor/spec_diff.py`. Addresses linkage drag (adversarial review ֲ§2): without section-level attribution, a beneficial mutation in one section can drag harmful mutations in adjacent sections into the archive.

### Key functions

**`parse_sections(spec_text)`** ג†’ `{section_title: content}`
- Splits on `## ` headings; pre-heading content goes into `__preamble__`
- Sub-sections (`### `) remain within their parent `## ` section

**`frozen_section_names(spec_text)`** ג†’ `set[str]`
- Detects which `## ` sections fall inside `<!-- BEGIN/END FROZEN: ... -->` blocks
- Used to flag dangerous mutations that touch FROZEN content

**`diff_spec(spec_a, spec_b)`** ג†’ `SpecDiff`
- Generates `difflib.unified_diff`
- Parses each hunk to attribute added/removed lines to `## ` sections via `_build_section_ranges` + `_section_for_line`
- Returns: `sections_changed: list[SectionChange]`, `sections_unchanged`, line counts, `unified_diff` string

**`attribute_fitness_delta(spec_diff, summary_before, summary_after)`**
- Computes `viability_rate_delta`, `fitness_mean_deltas` (per metric), `sections_changed`, `entanglement_score` (sections_changed / total_sections), `frozen_sections_changed`
- Entanglement score: 0.0 = single-section mutation, 1.0 = entire spec touched

**`apply_spec_diff(original, unified_diff)`** and **`revert_spec_diff(modified, unified_diff)`**
- Custom unified diff parser + applier (`_parse_unified_diff_hunks`, `_apply_hunk`)
- `reverse=True` swaps `-`/`+` roles for revert
- Maintains offset accounting for multi-hunk diffs

**Test results:** 18 new tests across `TestParseSections`, `TestDiffSpec`, `TestAttributeFitnessDelta`, `TestApplyRevertDiff`. 232 total pass.
