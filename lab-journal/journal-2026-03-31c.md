## cambrian-v3w: Entanglement Monitor (`supervisor/entanglement.py`)

**Purpose:** Track whether spec mutations are touching more sections simultaneously over time (rising entanglement = modularity degradation). **Design:** Pure computation â€” no LLM, no I/O. Consumes `list[SpecDiff]` from `spec_diff.diff_spec` and produces metrics.

**Key data structures:**

## cambrian-3qo: Adaptive Test Generation (`supervisor/adaptive_tests.py`)

**Purpose:** After each failed campaign, extract the dominant failure mode and generate 3-5 targeted pytest test cases probing that exact failure. Inspired by Absolute Zero Reasoner (arXiv 2505.03335) â€” self-generated tests as co-evolutionary pressure orthogonal to fixed evaluation curriculum.

**Key constants:**
- `EXPIRE_AFTER = 5` (campaigns): each adaptive test expires 5 campaigns after creation
- `MAX_ACTIVE = 10`: cap to prevent bloat
- `TESTS_PER_GENERATION = 4`: tests requested per generation round

**Storage:** `adaptive-tests.json` in the artifacts repo. Each record: `{test_id, test_code, failure_stage, campaign_id, created_at_campaign, expires_after, created_at}`.

**Expiry semantics:** `campaign_index < created_at_campaign + expires_after` â€” wall-clock independent, correct across paused/resumed evolution runs.

**Functions:**
- `load_tests / save_tests` â€” JSON persistence
- `get_active_tests(campaign_index)` â€” unexpired tests, newest first, capped at MAX_ACTIVE
- `expire_old_tests(campaign_index)` â€” remove expired, return count removed
- `generate_adaptive_tests(campaign_summary, spec_text, campaign_index, ...)` â†’ `list[dict]`
  - Skips if `viability_rate >= 1.0` or no non-none failures
  - Identifies `dominant_stage = max(non_viable, key=count)`
  - Calls LLM (default: `claude-sonnet-4-6`) with stage-specific guidance prompt
  - Appends to existing tests and saves
- `_build_test_gen_prompt(...)` â€” stage guidance for manifest/build/test/start/health failures
- `_extract_test_cases(response_text)` â€” splits on `def test_` boundaries; strips markdown fences
