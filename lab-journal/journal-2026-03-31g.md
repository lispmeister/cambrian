## Phase 1: M2 First Run Attempt

  CAMBRIAN_MINI_CAMPAIGN_N=1 CAMBRIAN_BO_INITIAL_POINTS=3 \

### Immediate failure: non-streaming LLM call

```
error: generate_artifact_failed
  'Streaming is required for operations that may take longer than 10 minutes'
```

**Root cause:** `prime_runner.py` used `client.messages.create()` instead of `client.messages.stream()`. CAMBRIAN-SPEC-005 §LLM Integration line 453 explicitly mandates streaming. The spec rule existed since cambrian-ux4 (2026-03-29) but was not followed when prime_runner.py was written for M2.

**Fix (cambrian-9kn):** Switched to `async with client.messages.stream(...) as stream: response = await stream.get_final_message()`.

### Audit of all LLM call sites

Found two more non-streaming calls:
- `adaptive_tests.py:165` — `client.messages.create()`
- `spec_mutator.py:79` — `client.messages.create()`

**Fix (cambrian-i8o):** Switched both to streaming. Updated 3 test mocks from `messages.create` to `messages.stream` async context manager pattern.

### Campaign artifact path fix

Also committed fix from prior session for campaign artifact naming:
- Double-prefix bug: `campaign-campaign-xxx-gen-N` → `campaigns/{id}/gen-{N}`
- Non-viable artifact cleanup via `shutil.rmtree`

## Key Observations

1. **The streaming bug is a recurring pattern.** This is the third time it has surfaced (cambrian-svf in gen-1, cambrian-ux4 in gen-2, now cambrian-9kn/i8o in M2 infrastructure). The spec rule was added after the first occurrence but new code kept missing it. The spec now has a prominent MUST rule at line 453.

2. **prime_runner.py was written without consulting the spec's LLM Integration section.** Multiple deviations (streaming, model escalation, parse repair, informed retry, token budget semantics) all trace to the same root cause: the M2 bridge was implemented from intent rather than from spec.

3. **Generation record schema has systematic null-vs-absent mismatch.** Python's `None` serializes to JSON `null`, but the spec says MAY fields should be absent (key omitted), not null. This affects `artifact-ref`, `completed`, and `viability`.
