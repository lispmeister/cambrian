## What Changed and Why

### Fix 1: CRITICAL — `kw.keyword` → `kw.arg` (test_rig.py:332)

`ast.keyword` nodes have `.arg` (the keyword name as a string), not `.keyword`. The original code would crash with `AttributeError` on any log call with keyword arguments, making the structlog lint gate 100% ineffective — it would kill the pipeline on first encounter instead of flagging the anti-pattern.

Verified: `python3 -c "import ast; kw = ast.parse('f(x=1)').body[0].value.keywords[0]; print(hasattr(kw, 'keyword'))"` → `False`

Impact: The entire 5-10% viability improvement from the structlog gate was nullified by this bug. This is the most important fix in this session.

### Fix 2: Expand printf-format regex (test_rig.py)

Original: `any(fmt in pos_args[0].value for fmt in ("%s", "%d", "%r"))` — misses `%f`, `%i`, `%x`, `%o`, `%e`, `%g`, and width specifiers like `%02d`.

Fixed: `re.compile(r"%[-+0 #]*\d*\.?\d*[diouxXeEfFgGcrsab]")` — covers the full printf spec.

### Fix 3: Add `entry.start` module-form exemplar (prime_runner.py)

The #1 historical failure mode (gens 20-29: 10 consecutive failures with `python src/prime.py`) had a rejection gate but no exemplar. Prevention (correct pattern shown) is better than rejection (correct pattern inferred from error). Added both to the CORRECT PATTERNS section and the DO NOT list.

### Fix 4: Guard baseline contract `KeyError` (test_rig.py:938)

`name = contract["name"]` was a bare dict access. A malformed historical manifest (missing `name`, `path`, or `expect`) would crash all future baseline checks with `KeyError`. Changed to `contract.get("name")` with validation and a log warning, skipping malformed contracts rather than crashing.

### Fix 5: Scope `force=True` to `append_only` fields (generations.py)

`force=True` bypassed the terminal-state guard for ALL fields — including `outcome` and `viability`. A bug passing extra keys to `run_baseline_reverse_run` could clobber critical terminal-state data.

Added `append_only: frozenset[str] | None = None` parameter. Updated the one call site to use `append_only=frozenset({"baseline-reverse-run"})`. This scopes the post-terminal write to exactly the annotation fields it needs.

### Fix 6: Log warning on `baseline.extract()` failure (supervisor.py)

`baseline.extract()` returned `None` on failure, but `handle_promote()` ignored the return value. Promotion would succeed while silently losing the baseline. Now logs `baseline_extract_failed` if extraction returns `None`.

### Fix 7: Empty contracts return `None`, not false-positive `passed=True` (test_rig.py)

When a baseline has no contracts, the original code returned `{"passed": True, "duration_ms": 0, "contracts": {}}`. This caused `baseline_contract_pass_rate = 1.0` via the fitness computation's `else 1.0` branch — indistinguishable from "all contracts passed."

Changed: return `None` instead. The `if baseline_check is not None` guard at the call site (line 1439) means the metric is simply absent from the fitness vector. Selection policies get no signal rather than a misleading signal.
