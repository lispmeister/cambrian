## Critical Issues

### 1. Prime never called /promote or /rollback (cambrian-ye2)

The most dangerous bug. The spec says Prime decides the outcome — viable → promote, non-viable → rollback. But Gen-1's generation loop just read the polling result and moved on. The `supervisor.promote()` and `supervisor.rollback()` functions existed but were dead code.

This worked because the Supervisor auto-promoted in `run_test_rig()`. But it created a fragile asymmetry: if Gen-2 implemented the spec faithfully (calling promote), the Supervisor would have already done it → double promote → git error.

**Fix:** Two-sided change.
- Supervisor: `run_test_rig()` now sets outcome to `"tested"` (not `"promoted"/"failed"`). No git ops.
- Prime: reads viability from the record, then explicitly calls `/promote` or `/rollback`. These endpoints do the git operations and set the terminal outcome.
- `poll_until_terminal` changed from checking specific terminal states to checking `outcome != "in_progress"`.

This is the correct ownership model: the Supervisor runs the pipeline and reports results. Prime makes the decision.

### 2. Viability report schema mismatch (cambrian-xdu)

The Test Rig wrote `checks.build.status = "pass"` but the spec says `checks.build.passed = true`. Also missing MUST fields: `generation`, `failure_stage`, `completed_at`. Missing `manifest` and `start` checks entirely.

If Gen-2 tried to parse the viability report per spec, it would find wrong field names.

**Fix:** Rewrote the Test Rig pipeline to match the spec schema exactly. Added manifest check, start check (detects immediate crashes), structured diagnostics on every failure.

## High Issues

### 3. Spec version frontmatter vs footer mismatch (cambrian-239)

Two version strings in the spec: frontmatter said 0.5.0, footer YAML said 0.5.1. The version bump in the previous session updated one but not the other.

**Fix:** Aligned frontmatter to 0.5.1.

### 4. Missing stats-generation contract (cambrian-dll)

Spec shows 3 contracts in the manifest. Gen-1 only emitted 2 — missing `stats-generation` with `body_contains: {generation: N}`.

**Fix:** Added the third contract to `build_manifest()`.

### 5. compute_artifact_hash ignored file paths (cambrian-6bl)

Only hashed file contents, not paths. Renaming a file without changing content → identical hash. Violates the intent of content-addressing.

**Fix:** `hasher.update(f.encode())` before `hasher.update(file_path.read_bytes())`.

### 6. Missing test_supervisor.py (cambrian-nic)

Spec requires tests for: supervisor client, generation number computation, retry logic. Gen-1 had no `test_supervisor.py` and no integration tests at all.

**Fix:** Added `test_supervisor.py` with 7 tests using a mock aiohttp server. Tests cover: get_versions, spawn, promote, rollback, poll_until_terminal (immediate and delayed).
