## Phase 1: P1 Beads (cambrian-rc8, cambrian-80h, cambrian-omj, cambrian-axo)

All four were implementation gaps identified in the 2026-03-31e audit ‚Äî code that was specced but missing.

### cambrian-rc8 ‚Äî Artifact-hash computation

`artifact-hash` was hardcoded to `""` in `handle_spawn` (supervisor.py:186). This violated CAMBRIAN-SPEC-005 and BOOTSTRAP-SPEC-002 ¬ß1.3 which declare it MUST.

**Fix:** Added `compute_artifact_hash(artifact_root, files)` to `supervisor.py` following the exact algorithm from CAMBRIAN-SPEC-005:

```python
def compute_artifact_hash(artifact_root: Path, files: list[str]) -> str:
    hasher = hashlib.sha256()
    for rel_path in sorted(files):          # lexicographic sort required
        if rel_path == "manifest.json":
            continue                        # excluded per spec
        hasher.update(rel_path.encode())
        hasher.update(b"\0")               # null separator ‚Äî critical for collision resistance
        hasher.update((artifact_root / rel_path).read_bytes())
    return f"sha256:{hasher.hexdigest()}"
```

Called in `handle_spawn` after reading `manifest.json` from the artifact. Gracefully degrades to `""` if manifest is absent or malformed (logs a warning).

### cambrian-80h ‚Äî Campaign runner promote/rollback

`run_campaign()` polled until generation completed but never called `/promote` or `/rollback`. Generations accumulated in `tested` state indefinitely; git branches never reached terminal state.

**Fix:** Added `_promote_or_rollback()` to `campaign.py`. Called after each generation's outcome leaves `in_progress`:

- `viability.status == "viable"` ‚Üí POST `/promote`
- otherwise ‚Üí POST `/rollback`
- Already-terminal outcomes (`promoted`, `failed`, `timeout`) skipped

Returns the updated record with the terminal outcome set.

### cambrian-omj ‚Äî Adaptive tests integration

`generate_adaptive_tests()` and `expire_old_tests()` in `adaptive_tests.py` were dead code ‚Äî never called.

**Fix:** Wired into `bo_loop.py`:
- `expire_old_tests(campaign_index)` at the start of each BO iteration
- `generate_adaptive_tests(full_summary, spec_text, campaign_index)` after each failed full campaign (`full_viability < 1.0`)

### cambrian-axo ‚Äî M2 entry point

No runnable script existed to launch the M2 evolution loop.

**Fix:** Created `scripts/run_m2.py`:
- Loads `.env` from project root before importing supervisor modules
- Resolves spec path from `argv[1]`, `CAMBRIAN_SPEC_PATH`, or default `spec/CAMBRIAN-SPEC-005.md`
- Instantiates `SpecBOLoop` with env-var-configurable parameters
- Calls `loop.run()`, prints `BOResult`, writes best spec to `best-spec.md`

Usage: `python scripts/run_m2.py [spec_path]`


Returns 409 Conflict. The existing duplicate-detection logic in `cambrian-gey` from the previous session (checking `generations.get` before spawning) combined with the new `fcntl` file locking (cambrian-tch) closes the race fully.

## Phase 3: P3 Beads (cambrian-tif, cambrian-uf0, cambrian-0x6, cambrian-tch)

### cambrian-tif ‚Äî Dockerfile: COPY full test-rig directory

Dockerfile had:
```dockerfile
COPY test-rig/test_rig.py /test-rig/test_rig.py
```

BOOTSTRAP-SPEC-002 ¬ß3.1 specifies `COPY test-rig/ /test-rig/`. If any supporting files are added to `test-rig/` (e.g. fixtures, helpers), the single-file COPY would silently exclude them.

**Fix:** `COPY test-rig/ /test-rig/`

### cambrian-uf0 ‚Äî Git concurrency control

`git_ops.py` had no concurrency guard. Concurrent `/spawn` calls could interleave `checkout`, `branch`, `add`, `commit` across generations, corrupting the artifacts repo state.

**Fix:** Module-level `_git_lock: asyncio.Lock`. The multi-step spawn sequence extracted from `handle_spawn` into a new `create_generation_branch(generation)` function that holds the lock for the entire sequence. `promote()` and `rollback()` also hold the lock for their sequences. `git()` itself does not acquire the lock ‚Äî callers at the sequence level own it.

All three test fixture `mock_git_ops` definitions (in `test_supervisor.py`, `test_lifecycle.py`, `test_security.py`) updated to include `create_generation_branch = AsyncMock()`.

### cambrian-0x6 ‚Äî completed timestamp on terminal outcomes only

`generations.update()` set `completed = datetime.now()` on every call:

```python
if "completed" not in fields:
    record["completed"] = datetime.now(UTC).isoformat()
```

This stamped `completed` when attaching a viability report (outcome=`tested`), which is non-terminal. The Supervisor later calls `/promote` or `/rollback` ‚Äî but `completed` was already set to the wrong time.

**Fix:** Only set `completed` when the record's outcome transitions to a terminal state:

```python
new_outcome = record.get("outcome")
if new_outcome in _TERMINAL_OUTCOMES and not record.get("completed"):
    record["completed"] = datetime.now(UTC).isoformat()
```

Updated `test_update_sets_completed_timestamp` (√ó2, in `test_supervisor.py` and `test_security.py`) to verify the new semantics: `tested` ‚Üí no `completed`; `promoted` ‚Üí `completed` set.

### cambrian-tch ‚Äî File locking on generations.json

`load_all`, `append`, and `update` performed unguarded read-modify-write. Two test rig tasks completing simultaneously could lose one update.

**Fix:** Replaced plain file I/O with two primitives:

- `_read_locked(path)` ‚Äî `fcntl.LOCK_SH` for reads
- `_atomic_modify(path, modifier)` ‚Äî `fcntl.LOCK_EX` held across read-modify-write; `modifier` returns `None` to signal no-op (skips write, preserves mtime)

The no-op return path was critical: `test_update_nonexistent_generation_is_noop` asserts the file's mtime is unchanged when updating a missing generation. Without it, `_atomic_modify` would write the file back unchanged, bumping mtime and failing the test.
