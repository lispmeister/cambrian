## Phase 2: uv in Docker Containers (cambrian-luv)

**Problem:** The project mandates `uv` everywhere on the host but `docker/Dockerfile` was using raw `pip install`. Inconsistency: host and container used different package managers.

**Fix:**

```dockerfile
# Before
RUN python -m venv /venv
ENV PATH="/venv/bin:$PATH"
RUN pip install --no-cache-dir aiohttp pytest pytest-asyncio pytest-aiohttp

# After
COPY --from=ghcr.io/astral-sh/uv:latest /uv /usr/local/bin/uv
RUN uv venv /venv
ENV PATH="/venv/bin:$PATH"
RUN uv pip install --no-cache-dir aiohttp pytest pytest-asyncio pytest-aiohttp
```

The uv binary is copied from Astral's official Docker image — no `apt install`, no version pinning drift.

**Spec updates:**
- `spec/CAMBRIAN-SPEC-005.md` line 142: manifest build example `pip install -r requirements.txt` → `uv pip install -r requirements.txt`
- `spec/BOOTSTRAP-SPEC-002.md`: §"Always use virtual environments" now mandates uv in containers; build stage timeout comment updated; §3.1 Dockerfile block updated to match
