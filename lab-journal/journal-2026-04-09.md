## Analysis: What's Actually Causing the 20% Failure Rate

Reading the full journal arc (39 entries, gens 1-48) to find the real failure pattern:

| Category | Count | % of LLM-attributable failures |
|----------|-------|---------------------------------|
| structlog anti-patterns | high | gens 17, 43, 47 explicit; 6/8 viable gens have minor forms |
| Test import discipline | medium | gen 19: missing `make_app` import in one test |
| `</file:end>` parser bug | 2 | gens 5, 7 — repeated identical bug |
| `time.time()` vs `time.monotonic()` | 6/8 | visible in viable gens as a quality issue |
| No exponential backoff | 7/8 | only gen-42 had it right |
| Python 3.14 syntax | low | potential failure class, caught by pytest collection errors |

The key insight from the journal: **prose guidance in the spec does not reliably change LLM behavior**. Gens 20-29 failed 10 times with the same `python src/prime.py` bug despite the spec saying the opposite — because the bug was in trusted infrastructure code (prime_runner.py line 146), not in the LLM output. The pattern holds for LLM-attributable failures too: structlog guidance was added in v0.14.3 but gens 43 and 47 still produced the anti-pattern.

Two mechanisms actually work:
1. **Mechanical gates** that catch bad code before it reaches runtime (the manifest gate catching script-form start commands is the paradigm case)
2. **Code exemplars** in the system prompt (LLMs reproduce correct patterns much better than they follow prose rules)

## Implementation

Five changes across 4 files, ~140 LOC total.

### 1. Docker: pre-install core deps + PYTHONPATH

`docker/Dockerfile`:
- Added `anthropic pydantic` to the pre-install line. Every viable gen needs these; pre-installing means a `requirements.txt` typo won't kill the build at Stage 2.
- Added `ENV PYTHONPATH=/workspace`. Belt-and-suspenders for intra-package imports in edge cases that slip past the manifest validation gate.

### 2. System prompt: code exemplars + DO NOT list

`supervisor/prime_runner.py` SYSTEM_PROMPT expanded from 30 lines to ~75 lines:

Added "CORRECT PATTERNS — copy these exactly" section with working code for:
- structlog: `log.info("prime_starting", generation=1)` — no `event=` kwarg
- Uptime: `time.monotonic()` not `time.time()`
- aiohttp test: local import of `make_app` inside each test function
- LLM streaming: `client.messages.stream()` pattern
- Supervisor backoff: exponential with `aiohttp.ClientError`

Added "DO NOT" list with 7 specific anti-patterns from distiller analysis.

Rationale: LLMs reproduce correct code they've seen; prose instructions are weak. The spec contains all of this but it's buried in 920 lines. The system prompt has priority attention.

### 3. Test Rig: syntax compilation gate

`test-rig/test_rig.py` — new `run_syntax_check()`:
- Runs `ast.parse()` on every `.py` file in `/workspace` between build and test.
- Catches Python 3.14 SyntaxErrors (unescaped newlines, etc.) with clear `file:line` diagnostics instead of confusing pytest collection errors.
- On failure, reports under `failure_stage: "build"` so the diagnostic feeds back to the next gen's `failed_context`.

### 4. Test Rig: structlog lint gate

`test-rig/test_rig.py` — new `run_structlog_lint()`:
- AST-walks all `.py` files under `/workspace/src/`.
- Detects two anti-patterns:
  1. `log.*(string, event=...)` — passes `event` twice → runtime TypeError
  2. `log.*(format_string_with_%s)` — structlog does not interpolate
- Reports `file:line` with the correct form.
- Runs between build and test, reports under `failure_stage: "build"`.

Turns a runtime TypeError that produces an opaque start-stage crash into an actionable pre-test diagnostic. This is the #1 LLM failure mode.

### 5. Campaign script: failed_context threading

`scripts/run_gen0_campaign.py` — mirrors the `failed_context` pattern from `supervisor/campaign.py`:
- After each failed gen, builds `failed_context` from the viability report diagnostics.
- Passes it to the next `prime_runner.generate_artifact()` call.
- Keeps the failed artifact dir alive until the next gen's prompt is built, then cleans up.

Previously `run_gen0_campaign.py` started each gen cold. This is the script used for all campaigns. Without feedback, the LLM can't correct known failures.

Also fixed stale test: `test_container_lifecycle_methods` was asserting `container.delete.assert_called_once()`, but `cambrian-p0z0` intentionally removed the `delete()` call to keep containers alive for log inspection. Updated to `assert_not_called()`.
