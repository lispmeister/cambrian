## 1. Infrastructure Bugs Found and Fixed

### 1.1 Streaming requirement (cambrian-svf)

Gen-1's `generate.py` used `client.messages.create()` with `max_tokens=32768`. The Anthropic
SDK rejected this with:

```
Streaming is required for operations that may take longer than 10 minutes.
```

**Fix:** Changed `call_llm()` to use `async with client.messages.stream() as stream:` with
`await stream.get_final_message()`. This is the correct pattern for large outputs; the spec
already stated "Streaming MUST be used" (BOOTSTRAP-SPEC-002 §… the system prompt note) but
the gen-1 implementation missed it. Fixed in `gen-1-new/src/generate.py` and manifest hash
updated.

### 1.2 Git state bug: `generations.json` branch conflict

The Supervisor's spawn handler creates a branch (`gen-N`), commits the artifact with
`git add -A`, and stays on that branch. When the test rig finishes and the next spawn is
called, `ensure_on_main()` runs `git checkout main` — which fails:

```
error: Your local changes to the following files would be overwritten by checkout:
    generations.json
```

Root cause: `generations.json` was committed on the `gen-N` branch (via `git add -A`), then
updated by the Supervisor after the test run. Checking out `main` would overwrite those
uncommitted updates.

**Fix:** Added `generations.json` to `.gitignore` in the artifacts repo and ran
`git rm --cached generations.json` to untrack it. Now:

- `git add -A` never stages `generations.json` (ignored)
- Branch switches never conflict on it — it persists in the working tree untouched
- The file accumulates generation history without git tracking, managed entirely by the
  Supervisor process

This also means `git branch -D gen-N` in the rollback path now works (we return to main
cleanly before the next spawn).

BOOTSTRAP-SPEC-002 does not document this requirement; a follow-up patch will add it.

## 3. Root Cause: Python 3.14 String Literal Restrictions

The LLM consistently generated test code like this:

```python
close_tag = "
</file>"
```

Python 3.14 (following the deprecation in 3.12) treats an unescaped newline inside a
regular string literal as a `SyntaxError: unterminated string literal`. The correct form
is either:

```python
close_tag = "\n</file>"      # escaped newline
close_tag = """
</file>"""                   # triple-quoted
```

This failure appeared in `tests/test_generate.py`, which tests the `parse_files()` function.
That function extracts `<file path="...">content</file>` blocks from LLM responses. Writing
test strings for this naturally involves multi-line file content, which is where the LLM
went wrong.

The retry prompt included the full error message and failed source code, but Sonnet 4.6
repeated the same mistake across three separate retries. The model either lacks training
data on Python 3.14's strictness or doesn't connect the error message to the fix.

## 4. Genome Gaps Identified

### 4.1 System prompt underspecifies Python 3.14

The SYSTEM_PROMPT says:

```
- The code must work in Python 3.14 inside a Docker container with a venv at /venv.
```

This states the requirement but not the constraints. "Works in Python 3.14" doesn't tell the
LLM what Python 3.14 enforces more strictly than 3.11 or 3.12. The spec needs to encode the
environmental fitness constraints precisely enough that the LLM can implement them correctly.

### 4.2 System prompt / spec divergence risk

The SYSTEM_PROMPT in `generate.py` is a hard-coded string, but it also appears verbatim in
CAMBRIAN-SPEC-005. These can diverge across generations. If the spec is updated but
offspring inherit the old system prompt from the parent's source code, the fix won't
propagate. For now both must be kept in sync manually; a future improvement is to have the
spec be the sole source of truth.

### 4.3 No retry escalation policy

After 3 identical failures, the system did nothing different. The spec doesn't define when
to escalate (use a stronger model, change the prompt strategy, or give up). This is an M2
concern — the organism should eventually be able to observe its own failure patterns and
mutate its genome to avoid them.
