## Phase 2: M2 Launch — Bug Hunt

### Bug 1: spec_mutator _MAX_TOKENS too small

**Symptom:** Every mutation attempt failed with `missing_required_section` ×2 + `frozen_block_removed`. Three retries, exhausted, iteration skipped. Happened on every section target.

**Root cause:** `_MAX_TOKENS = 8192`. The spec is ~13,271 tokens. The LLM physically cannot output the complete spec — it truncates at ~8k tokens, dropping the tail (Implementation Requirements at line 444, Acceptance Criteria at 561, Verification Layers at 601, FROZEN block at 654–718). The grammar guard correctly rejects all of these.

**Fix:** Raise default to 32768 (matches `_PER_CALL_MAX_TOKENS` in prime_runner). Committed `f1bd901`.

### Bug 2: Docker image stale — two mismatches

**Symptom:** Containers spawned and exited in ~250ms without a viability report (`viability_report_missing`). User confirmed no container appeared in Docker Desktop dashboard.

**Investigation:**
```
docker run --rm cambrian-base:latest python -c "..."
# Output: [test-rig] Stage: manifest
#         [test-rig] Report written to /workspace/viability-report.json
```
The image (built 2026-03-27) predated two breaking changes:

| Change | Bead | Old behaviour | New behaviour |
|--------|------|---------------|---------------|
| Container isolation | cambrian-2cf | writes report to `/workspace` | writes to `/output` |
| Wire format | (field rename) | validates `created_at` | validates `created-at` |

The supervisor now mounts a separate `output_dir` at `/output` — the old image never wrote there, so the report was always "missing".

**Fix 1:** Rebuild image → report path corrected. (`./docker/build.sh`)

### Bug 3: /venv owned by root — build permission denied

**Symptom:** After image rebuild, containers ran but failed at build stage in ~3.7s:
```
error: Failed to install: typing_inspection-0.4.2-py3-none-any.whl
  Caused by: failed to create directory `/venv/lib/python3.14/site-packages/typing_inspection`
  Permission denied (os error 13)
```

**Root cause:** Dockerfile creates `/venv` and installs test rig deps as root, then switches to `cambrian` user. The `entry.build` command (`uv pip install -r requirements.txt`) runs as `cambrian` and cannot write to `/venv`.

**Fix:** `RUN useradd -m -s /bin/bash cambrian && chown -R cambrian:cambrian /venv`. Committed `7c6c555`.
