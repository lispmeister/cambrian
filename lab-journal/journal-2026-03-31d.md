### Category 2: Spec Says X, Code Does Y

**2a. BO loop vs MAP-Elites:** Spec describes MAP-Elites grid (240 cells, 4 dimensions). Code uses GP-EI single-objective BO via scikit-optimize. Rationale: adversarial review §4 — MAP-Elites needs 200+ evals, Stage 1 budget is ~20-30 campaigns.

**2b. Dual-model screening vs deterministic grammar:** Spec says "Use Sonnet for coherence screening." Code uses `spec_grammar.py` — required sections, port number, FROZEN block byte comparison, no LLM. Rationale: adversarial review §5 — LLM screening its own output is gameable.

**2c. `spec-hash` in `/spawn`:** Spec (BOOTSTRAP-SPEC-002 §1.2) says MUST. Code: `body.get("spec-hash", "")` treats as optional.
