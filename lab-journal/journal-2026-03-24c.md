## Decisions Made

### 1. Python 3.14 (not 3.14t) for M1

Free-threaded Python (3.14t) is not installed and is deferred to M2. M1 uses the standard Python 3.14 build.

**Spec changes:** CAMBRIAN-SPEC-005 § Implementation Requirements, BOOTSTRAP-SPEC-002 § Dockerfile, asyncio rationale, language metadata.

### 2. Separate artifacts repo

Generated artifacts go in a separate git repo (`cambrian-artifacts/`), not as branches in the Cambrian project repo. This keeps the project repo clean.

**New config variable:** `CAMBRIAN_ARTIFACTS_ROOT` (default: `../cambrian-artifacts`) — replaces the former `CAMBRIAN_WORKSPACE_ROOT`.

**Spec changes:** BOOTSTRAP-SPEC-002 § Git Operations, § Directory Structure, § Configuration, § HTTP API descriptions, § Validation.

### 3. Gen-0 convention

The test artifact is `gen-0/` in the artifacts repo. It is the zeroth generation: hand-crafted, not LLM-generated. All other generations follow the same `gen-N/` convention.

**Spec changes:** BOOTSTRAP-SPEC-002 § Test Artifact (new §4.2 Location), § Bootstrap Sequence, § Directory Structure.

### 4. 10-second health-check timeout

If Prime doesn't respond on `/health` within 10 seconds, the start stage fails. Previous spec said 30s.

**Spec changes:** CAMBRIAN-SPEC-005 § Failure Handling.

### 5. Supervisor stays on host

Docker doesn't eliminate the need for a host-side process. The Supervisor is the host-side orchestrator that manages Docker, git, and generation records. Not containerized in M1.

### 6. API key injection

The Supervisor reads `ANTHROPIC_API_KEY` from its own environment and injects it into Prime containers at launch. The key is never written to disk or committed to git.

### 7. Human-operated Prime launch (M1)

No `/start` endpoint in M1. The human operator runs a `docker run` command to start a Prime container. From that point, Prime operates autonomously.

### 8. Test artifact content

The gen-0 test artifact includes:
- An aiohttp server with `/health` (returns `{"ok": true}`) and `/stats`
- A pytest that hits `/health` and asserts 200

Pytest is sufficient to prove the test stage works. No elaborate test suite needed.

### 9. M1 acceptance condition

M1 is done when Gen-3 passes the Test Rig. The chain: human creates Gen-1, Gen-1 produces Gen-2 autonomously, Gen-2 produces Gen-3 autonomously, Gen-3 passes the Test Rig. Gen-3 does not need to actually spawn Gen-4.

Before moving to M2, human + Claude Code will analyze Gen-1, Gen-2, and Gen-3 to confirm stability.
