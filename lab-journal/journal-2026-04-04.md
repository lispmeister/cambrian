## Phase 2: Root Cause — Missing structlog in Docker Image

Container logs were unavailable (cleaned up on rollback). Diagnosed by running the image directly:

```
docker run --rm cambrian-base:latest python -c "import test_rig"
ModuleNotFoundError: No module named 'structlog'
```

**Root cause:** `test_rig.py` gained a `structlog` import during the 2026-04-03 P0–P2 fix sweep (`cambrian-ad3n`), but `structlog` was not added to the `docker/Dockerfile` pip install line. The Docker image built today included the updated `test_rig.py` but not the new dependency.

**Fix (`cambrian-7krd`, P0):** Added `structlog` to the `uv pip install` line in `docker/Dockerfile`:

```dockerfile
# Before:
RUN uv pip install --no-cache-dir aiohttp pytest pytest-asyncio pytest-aiohttp pyyaml
# After:
RUN uv pip install --no-cache-dir aiohttp pytest pytest-asyncio pytest-aiohttp pyyaml structlog
```

Rebuilt the image (`./docker/build.sh`). Verified with `docker run --rm cambrian-base:latest python -c "import test_rig"` — clean import confirmed (PermissionError on `/output` is expected without volume mounts).

---

## Phase 3: Container Retention for Diagnostics

The crash showed a gap: when a container exits without writing a report, there is no diagnostic trace. Container logs are destroyed on deletion.

**Fix (`cambrian-p0z0`, P1):** Removed `container.delete()` from the `finally` block in `run_test_rig()` (`supervisor/supervisor.py`). Containers named `lab-gen-N` now persist after the test rig exits and can be inspected with:

```bash
docker logs lab-gen-N
```

Manual cleanup: `docker rm lab-gen-N` or `docker container prune`.

Note: `create_or_replace()` handles the case where a same-named container exists (always incremented generation numbers make collisions unlikely).

Also opened `cambrian-u38m` (P1) to track capturing container logs into the artifact directory for fully automated diagnostics.
