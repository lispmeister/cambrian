# Lab Journal — 2026-04-04c: Phenotypic Distiller Script

## Session Goals
1) Design and implement a post-campaign phenotypic distiller that automates the manual differential code analysis from journal-2026-04-04b
2) Validate against the gens 39-48 campaign

---

## Motivation

In journal-2026-04-04b, a manual comparison of 8 viable gens found that only 2/8 used `time.monotonic()` and only 1/8 had exponential backoff. Those observations were hand-encoded into spec v0.14.4 as SHOULD rules. This is the Baldwin Effect — phenotypic excellence feeding back into the genome — but it required a human reading diffs across 8 codebases.

The distiller automates this loop so it can run after every campaign.

## Implementation: `scripts/distill_campaign.py`

**Approach:** AST-based pattern extraction (not string matching). Each generation's Python source files are parsed into an `ast.Module` and walked for specific patterns.

**Patterns tracked (positive):**
- `time.monotonic()` for elapsed measurement
- Exponential backoff (`var * 2` or `var *= 2`)
- Specific exception types (not bare `except Exception`)
- Pydantic, pathlib, type annotations, dataclasses, docstrings

**Patterns tracked (negative / anti-patterns):**
- `time.time()` for elapsed measurement
- Bare `except:` or `except Exception`
- Structlog misuse (`event=` duplication, `event_msg=` non-standard kwarg)

**Two analyses per campaign:**
1. **Viable vs Failed** — patterns that correlate with passing the test rig
2. **Top-ranked vs Bottom-ranked viable** — patterns that correlate with higher fitness (composite score from test_pass_rate, test_count, spec_vector_pass_rate, assertion_density, plus bonus for monotonic/backoff)

**Output:** Markdown report with tables and proposed spec amendments including code examples extracted from the best gens.

## Validation: Gens 39-48

Ran against `gen-0-campaigns/gen0-1775283677` (the 10-gen campaign from journal-2026-04-04b). All 4 patterns identified in the manual analysis were detected automatically:

| Pattern | Top-ranked (4) | Bottom-ranked (4) | Delta |
|---------|:-:|:-:|:-:|
| time.monotonic() | 2/4 | 0/4 | +50% |
| Exponential backoff | 2/4 | 0/4 | +50% |
| time.time() (anti-pattern) | 2/4 | 4/4 | -50% |
| Structlog misuse | 0/4 | 2/4 | -50% |

The viable-vs-failed analysis was noisier (only 2 failed gens to compare against), confirming that the top-vs-bottom ranking of viable gens is the stronger signal source.

## Next Steps

To fully close the Baldwin Effect loop, feed the distiller's proposed amendments into `spec_mutator.py` as targeted mutation suggestions. This would replace random perturbation with informed mutation in the BO loop.
