## Phase 2: cambrian-4do — Layer 1 Spec-Vector Evaluation

## Phase 3: cambrian-2cf — Container Isolation Hardening

**Problem (adversarial review finding):** The organism and Test Rig share the same container with a single bind-mounted volume (`/workspace`). Under M2 selection pressure, evolved organisms could:
1. Write a fake `viability-report.json` into `/workspace` before the Test Rig does
2. Daemonize child processes that continue running after the Test Rig terminates them

Both vectors allow an organism to cheat its way to a `viable` outcome.

### Fix 1: Isolated output mount

The Supervisor now creates a randomly-named temp directory (`tempfile.mkdtemp(prefix="cambrian-output-")`) and bind-mounts it at `/output` inside the container. The Test Rig reads `CAMBRIAN_OUTPUT_DIR` (default: `/output`) and writes `viability-report.json` there.

**Why this works:** The mount source path is randomized at container creation time. Organism code running in `/workspace` has no way to predict or enumerate the host-side temp path. Even if the organism reads the baked-in `test_rig.py` to learn about `/output`, it cannot determine the host path needed to write a fake report before mount time.

The Supervisor reads the report from the temp dir after the container exits, then `shutil.rmtree()` cleans it up.

### Fix 2: Process group isolation

All subprocess calls in the Test Rig (`run_build`, `run_tests`, `start_process`) now use `start_new_session=True`. This places each subprocess in a new POSIX process group.

New helper `_kill_process_group(proc)`:
- Gets the process group ID via `os.getpgid(proc.pid)`
- Sends `SIGTERM` to the whole group
- Waits up to 5 seconds for graceful exit
- Sends `SIGKILL` if not exited

This ensures daemonized children (double-fork, `daemon=True`, etc.) are reaped when the Test Rig terminates the group.

### Fix 3: Spec vectors pre-read before build

An organism could modify the spec file during the build or test stages. To prevent this, spec vectors are now extracted immediately after the manifest stage (before build) and cached. The cached vectors are passed to `run_health_check()` at the health stage.

```python
# Pre-read spec vectors after manifest stage (before any organism code runs)
preread_spec_vectors: list[dict[str, Any]] | None = None
spec_file = _find_spec_file(manifest)
if spec_file is not None:
    preread_spec_vectors = _extract_spec_vectors(spec_file.read_text(encoding="utf-8"))
```
