## Work Completed

### 1) M2 Stage 1 success checklist added

Created `docs/M2-STAGE1-SUCCESS-CHECKLIST.md` with:

- pre-run invariants and fixed-run knobs
- required campaign sample size (baseline vs mutation)
- GO / INCONCLUSIVE / NO-GO thresholds
- hard fail conditions
- stability re-run requirements
- evidence/decision requirements

The checklist now includes a command to generate campaign and aggregate tables directly from run history.

### 2) Results template added

Created `docs/templates/m2-stage1-results-template.md` with:

- run metadata block
- per-campaign table format
- aggregated comparison table format
- top-3 stability rerun section
- invariant check checklist
- final decision block

### 3) Auto-summary script added

Created `scripts/summarize_m2_results.py`.

Behavior:

- reads generation records from `generations.json`
- groups by `campaign-id`
- computes per-campaign metrics:
  - viability rate
  - median viable `total_duration_ms`
  - mean `test_count`
  - mean `test_pass_rate`
  - mean `spec_vector_pass_rate`
  - mean `baseline_contract_pass_rate`
  - dominant failure stage
- computes aggregated baseline vs mutation comparison
- prints markdown tables suitable for direct paste into the results template
- optionally writes markdown and JSON outputs via `--markdown-out` and `--json-out`

CLI behavior notes:

- requires at least one `--baseline-campaign-id`
- accepts repeated `--baseline-campaign-id` / `--mutation-campaign-id`
- if mutation IDs are omitted, all non-baseline campaign IDs are treated as mutation group

### 4) README updated

Updated `README.md` to reference the new planning artifacts and summary helper script.
