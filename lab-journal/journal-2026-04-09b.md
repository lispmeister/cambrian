## Work Done

### cambrian-u38m: Container log capture before rollback

In `supervisor/supervisor.py`, `run_test_rig()`:
- After `container.wait()` completes (normal path): calls `container.log(stdout=True, stderr=True)` and writes to `artifact_path / "container.log"`.
- Timeout path: same capture after `container.kill()`, before returning.
- Both paths wrapped in `contextlib.suppress(Exception)` so log capture failure never blocks the pipeline.

The `run_gen0_campaign.py` already copies the artifact dir to `failed/` on failure, so `container.log` is automatically preserved there too.

### cambrian-a0n4: Baseline battery extraction

New module `supervisor/baseline.py`:
- `extract(generation, artifact_path, record)`: after a generation is promoted, copies contracts from `manifest.json` + metadata to `{artifacts_root}/baselines/gen-{N}/battery.json`.
- `latest()`: scans baselines dir, returns the highest-N battery or None.
- `battery_path_for(generation)`: returns path for a given generation (may not exist).

Hook in `supervisor/supervisor.py` `handle_promote()`: calls `baseline.extract()` after the generation record is updated to "promoted".

### cambrian-p1ha: Dual-run Test Rig

In `test-rig/test_rig.py`:
- New `BASELINE_PATH` constant (reads `CAMBRIAN_BASELINE_PATH` env var, default `/baseline/battery.json`).
- New `run_baseline_check(health_url, generation)`: loads battery.json, evaluates baseline contracts against the running server, returns results dict or None.
- Called in `run_pipeline()` AFTER `run_health_check()` but BEFORE `_kill_process_group(proc)` — the server must be live during evaluation.
- Results appended to `health_result["baseline-contracts"]` (informational, does NOT affect viability).

In `supervisor/supervisor.py` `run_test_rig()`:
- Before creating the container config, calls `baseline.latest()`.
- If a battery exists, adds bind mount `{battery_path}:/baseline/battery.json:ro` and env var `CAMBRIAN_BASELINE_PATH=/baseline/battery.json`.

### cambrian-kzc6: Fitness delta computation

In `test-rig/test_rig.py` `compute_fitness()`:
- Reads `checks["health"]["baseline-contracts"]` if present.
- Computes `baseline_contract_pass_rate = passed_baseline / total_baseline`.
- Absent if no baseline or baseline has no contracts.
- Weight 1.0 (external measurement — Test Rig evaluates, not the organism).

### cambrian-acb5: Reverse-run regression guard

New `run_baseline_reverse_run(current_generation, battery)` in `supervisor/supervisor.py`:
- Spawns a Docker container with the PREVIOUS baseline artifact mounted at `/workspace`.
- Waits for it (same CONTAINER_TIMEOUT).
- Reads viability report from isolated output dir.
- Stores result in `generations.json` under `"baseline-reverse-run"` using `force=True` to bypass the terminal-state guard.

`generations.update()` now accepts `force: bool = False` to allow post-terminal annotations.

Triggered from `handle_promote()`:
1. `prev_battery = baseline.latest()` — capture BEFORE extracting new baseline.
2. `baseline.extract(...)` — extract the new baseline.
3. `asyncio.ensure_future(run_baseline_reverse_run(...))` — fire-and-forget background task.

### cambrian-b0sx: Spec updates

Added §2.13 "Counterfactual Baseline Battery (M3)" to BOOTSTRAP-SPEC-002 covering:

- 2×2 regression matrix.
- `CAMBRIAN_BASELINE_PATH` env var added to `_ALLOWED_UNDERSCORES` in compliance tests.
- `baseline_contract_pass_rate` fitness metric added to allowed list.
