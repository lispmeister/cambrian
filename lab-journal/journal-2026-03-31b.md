## cambrian-2e5: Spec Grammar (`supervisor/spec_grammar.py`)

**Purpose:** Deterministic mutation screening. Replaces the gameable LLM dual-model screener (adversarial review §5) with grep-based structural checks.

**Key constants:**
- `REQUIRED_SECTIONS` (15 sections): the sections Prime must find to do its job
- `FROZEN_SECTION_NAMES`: sections protected by FROZEN markers
- `REQUIRED_PORT = 8401`
- `NORMATIVE_KEYWORDS = {MUST, MAY, SHALL}`

**Functions:**
- `validate_spec(spec_text)` → list[GrammarViolation]: checks required sections, port, keyword presence, duplicate headings, paired FROZEN markers
- `validate_mutation(parent, child)` → list[GrammarViolation]: all validate_spec checks + byte-for-byte FROZEN block comparison via regex extraction
- `evolvable_sections(spec_text)` → list[str]: all ## sections not in FROZEN_SECTION_NAMES
- `is_valid_spec / is_valid_mutation`: convenience helpers

The frozen block check uses `re.DOTALL` to extract block content between markers; any delta triggers `frozen_block_modified` (fatal). Test helper `_make_valid_spec()` puts Invariants only inside the FROZEN block (not duplicated in the section list) to avoid triggering `duplicate_heading`.

**Tests:** 10 new (242 total).

## cambrian-7cc: Type 1 Spec Mutator (`supervisor/spec_mutator.py`)

**Purpose:** Generate grammar-constrained refinement mutations using Opus.

**`type1_mutate(spec_text, campaign_summary, target_section, model)`:**
1. Builds prompt with viability_rate, fitness_trend (word form), failure_distribution
2. Calls `anthropic.AsyncAnthropic().messages.create()`
3. Strips markdown code fences from response (`_extract_spec_from_response`)
4. Runs `validate_mutation(original, mutated)` — deterministic FROZEN + grammar check
5. Retries up to `CAMBRIAN_MUTATION_MAX_ATTEMPTS` (default 3) on fatal violations
6. Returns `None` if all attempts fail

**`propose_target_section(campaign_summary, evolvable)`:**
Maps `failure_distribution` dominant stage to spec section:
- `build` → "Implementation Requirements"
- `test` → "Acceptance Criteria"
- `health` → "Contracts"
- `start` → "Implementation Requirements"
- `manifest` → "Artifact Manifest"

**Model default:** `CAMBRIAN_MUTATION_MODEL` env var → `CAMBRIAN_ESCALATION_MODEL` → `claude-opus-4-6`. Budget-conscious runs set `CAMBRIAN_ESCALATION_MODEL=claude-sonnet-4-6`.

**Tests:** 9 new (includes async mock tests for retry logic and all-failures case).

## cambrian-yy5: Single-Objective BO Loop (`supervisor/bo_loop.py`)

**Purpose:** Maximize viability_rate across spec mutations using Gaussian Process + Expected Improvement.

**Design (adversarial review §4):**
- Stage 1 (~20-30 campaigns): single-objective BO, not MAP-Elites (needs 200+ evals)
- Feature space: per-section line-delta from base spec (one Real dimension per evolvable section)
- `skopt.Optimizer(base_estimator="GP", acq_func="EI")` wraps the GP-EI loop
- BO suggests feature vectors; we decode to target section and call type1_mutate

**`extract_features(spec_text, section_names, base_spec)`:**
For each evolvable section: `len(current_lines) - len(base_lines)`. Returns a float vector.

**`decode_suggestion(suggested_x, section_names, campaign_summary)`:**
Picks the section with the largest magnitude in the suggested feature vector.
Falls back to `propose_target_section` if all magnitudes < 1.0.

**`SpecBOLoop.run()`:**
1. Evaluate base spec (establishes 0-point baseline)
2. For each budget iteration:
   a. `optimizer.ask()` → suggested feature vector
   b. Decode to target section
   c. `type1_mutate()` → mutated spec text
   d. `run_campaign(n=2)` → mini-campaign screening
   e. If passes screen: `run_campaign(n=5)` → full campaign
   f. `optimizer.tell(features, -viability_rate)` (skopt minimizes)
3. Returns `BOResult` with best spec, all observations, iterations count

**Dependency added:** `scikit-optimize>=0.9` in pyproject.toml (numpy, scipy, sklearn).
skopt 0.10.2 resolves cleanly on Python 3.14.3.

**Tests:** 6 new (feature extraction, BO init, optimizer dimensions, decode_suggestion).
